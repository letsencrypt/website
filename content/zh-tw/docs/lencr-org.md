# `lencr.org` 是什麼？

`lencr.org` 是 Let's Encrypt 旗下的一個域名。 我們用它來存放憑證中所引用的資料。

# 我的電腦為什麼會從這個網站獲取資料？ 這是惡意的嗎？

不，`lencr.org` 網站上的資料絕非惡意。 裝置會連線到 `lencr.org`，是因為該裝置上的用戶端（例如瀏覽器或應用程式）先前連線到另一個網站，發現了 Let's Encrypt 憑證，正在驗證其有效性。 這是許多用戶端的正常流程。

我們無法斷言上述*另一個網站*是否存在風險。 如果您正在調查異常的網路活動，或許應該關注與 `lencr.org` 緊鄰的前一條連線。

客戶端與 `lencr.org` 之間的連線模式可能較為反常，沒有規律可循。 有些用戶端從來不會從該網站獲取資料，有些則只下載部分資料，或者為提高效率而快取部分資料，因此只會偶爾（首次需要時以及資料可能已過期後）建立連線。

# 這些資料的用途是什麼？

當用戶端軟體（例如瀏覽器或應用程式）連線到一個網站，而該網站出示憑證時，用戶端需要驗證這份憑證的真實性與有效性。 以下資料可以從各方面幫助用戶端達成此目的。

* `c.lencr.org` 提供了憑證撤銷清單 (CRL)，列出我們所頒發、尚未過期但已撤銷的所有憑證。

* `i.lencr.org` 提供了我們的"簽發者"中繼憑證副本，這些證書由我們的根憑證簽署，或由其他憑證頒發機構 (CA)"交叉簽署"。 用戶端可以利用此資料來確認"信任鏈"，從它正在驗證的終端實體證書，經由一個或多個中繼憑證，追溯到它所認可並信任的根 CA 憑證。

# 為什麼用 "`lencr.org`"？

我們過去曾使用較長的網址，例如 `http://example.int-x3.letsencrypt.org/`。 然而，在我們頒發[新的根憑證和中繼憑證][1]時，我們希望盡可能縮減它們的大小。 網際網路上每一次 HTTPS 連線（每天數十億次）都必須傳送一份憑證副本，因此每個位元組都至關重要。 我們選擇了 `lencr.org`，因為它與我們的名稱相似：**L**et's **ENCR**ypt。 其發音很像虛構的地區 [Lancre][]，在 Terry Pratchett 的_《碟形世界》_小說裡。

[1]: https://letsencrypt.org/2020/09/17/new-root-and-intermediates.html
[Lancre]: https://wiki.lspace.org/Lancre
